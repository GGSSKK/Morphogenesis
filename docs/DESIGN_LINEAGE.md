# 設計系譜 — 胚発生型GA論文から Morphogenesis へ

---

## 1. 理論的源流

GA_Playground は**胚発生型遺伝的アルゴリズム (Embryogenic GA)**の論文群に影響を受けて
構築されたプロトタイプである。

### 1.1 胚発生型GA とは

通常の遺伝的アルゴリズムは「遺伝子型 → 表現型」の直接マッピング:
- 遺伝子の各ビット = 表現型の各パラメータ

胚発生型GAでは「遺伝子型 → 発生プロセス → 表現型」の間接マッピング:
- 遺伝子 = **発生ルール**（条件-アクション対）
- ルールの適用が時間的・空間的に展開される
- 同じ遺伝子から、初期条件によって異なる表現型が生まれうる

### 1.2 GA_Playground が統合を試みた4つの理論

| 理論 | GA_Playground での実装 | 完成度 |
|---|---|---|
| **遺伝的アルゴリズム (GA)** | 遺伝子文字列のランダム生成 | △ 生成のみ。適応度評価・選択・交叉・突然変異は未実装 |
| **セルオートマトン (CA)** | 近傍参照 (N-1, N-2) → 状態更新 | ○ 核心部分。8条件×8アクション |
| **L-System** | 文字列ルールの逐次解釈 | △ ルール解釈は実装。再帰・分岐は構想のみ |
| **拡散反応 (RD)** | 閾値+勾配条件 → 乗算反応 → リズム生成 | ○ 暗黙的に実現。2閾値(0.5/1.0)×勾配条件 |

## 2. GA_Playground の2つのプロトタイプ

### 01Test: 物理レイヤー（運動の生成）

```
[頭部 Ease 往復] → [ConfigurableJoint 連鎖] → [波動的うねり伝播]
```

- **焦点**: 形態ではなく**運動**のプロシージャル生成
- **手法**: 物理シミュレーション（ジョイント角度制限+減衰）
- **結果**: 1つのパラメータ（頭部の往復運動）から蛇のうねりが**創発**する

### 02Test: 形態レイヤー（構造の生成）

```
[遺伝子文字列] → [条件-アクション解釈] → [体節のスケール/色パターン]
```

- **焦点**: 運動ではなく**形態**のプロシージャル生成
- **手法**: セルオートマトン的ルール解釈（拡散反応パターン）
- **結果**: ランダム遺伝子から予測不能だが自己相似的な形状・色パターンが**創発**する

### 未統合の構想

2つのプロトタイプは**別々のシーン**に分離されたまま終わっている。
統合された場合:

```
[遺伝子] → [形態生成(02Test)] → [物理連結(01Test)] → [形態に応じた運動の創発]
         ↑                                              ↓
         └──────── [適応度評価] ← [運動の有効性] ←──────┘
```

太い体節は重く、細い体節は軽い → 遺伝子が決定する形態パターンが物理的な運動特性を左右する → GAの適応度関数として「どれだけ速く/遠くに移動できるか」を評価 → 形態と運動の共進化

## 3. Morphogenesis プロジェクトの方向性

### 3.1 GA_Playground から引き継ぐ設計思想

1. **間接マッピング**: 遺伝子 ≠ 直接的な形状パラメータ。遺伝子 = 発生ルール
2. **近傍参照パターン生成**: セルオートマトン的な局所的ルールから大域的パターンが創発
3. **条件-アクション対**: 4bit 条件 + 4bit アクション の遺伝子エンコーディング
4. **乗算ベースの反応**: スケール変更は加算ではなく乗算（指数的ダイナミクス）
5. **クランプによる安定化**: [0.05, 8.0] の有限レンジ

### 3.2 GA_Playground の限界を超えるために

| 限界 | 解決方針 |
|---|---|
| 1次元のみ | **3次元空間**での体節配置。分岐構造の導入 |
| Xスケールのみ | **多次元状態**: スケールXYZ + 色 + テクスチャ + 密度 |
| 全体節に同一遺伝子 | **位置依存の遺伝子発現**: Hox遺伝子モデル（位置情報による差異化） |
| 生成時のみ | **ランタイム発生**: 連続的な成長・分化プロセス |
| GAの進化なし | **完全なGA**: 適応度評価 → 選択 → 交叉 → 突然変異 |
| L-Systemの再帰なし | **再帰的分岐**: 各体節がサブツリーを生成可能 |
| 物理未統合 | **形態-運動共進化**: 物理シミュレーションと形態生成の統合 |

### 3.3 技術スタックの候補

GA_Playground は Unity 5 + PlayMaker（ビジュアルスクリプティング）で構築されていた。
Morphogenesis では以下を検討:

| レイヤー | 候補技術 | 理由 |
|---|---|---|
| **発生ルールエンジン** | Python / Rust | 高速なルール解釈 + GA進化ループ |
| **3D形態生成** | Blender Geometry Nodes / SDF | プロシージャルメッシュ生成 |
| **物理シミュレーション** | Unity / Godot / JAX-MD | 物理ベースの運動テスト |
| **GA進化** | DEAP / custom | 遺伝子操作 + 適応度評価 |
| **可視化** | Three.js / Blender | リアルタイム or オフライン |

## 5. GA_Playground 遺伝子エンコーディングの拡張案

### 5.1 現行（GA_Playground）

```
遺伝子 = [Cond:4bit][Action:4bit] × 20ルール
条件: 8種（閾値比較 + 勾配比較 + ワイルドカード）
アクション: 8種（スケール乗算5種 + マテリアル2種 + Terminate）
操作対象: Xスケールのみ
空間: 1次元
```

### 5.2 拡張案（Morphogenesis）

```
遺伝子 = ヘッダー + [Cond:8bit][Action:8bit][Target:4bit] × N ルール
条件: 拡張
  - 近傍参照: N-1, N-2, ..., N-k（k可変）
  - 多次元状態比較: scaleXYZ, color, density
  - 位置情報: 頭からの距離、分岐深度
  - ランダム確率: P(fire) = 0.0〜1.0
アクション: 拡張
  - 乗算: ×0.5, ×0.9, ×1.0, ×1.1, ×2.0
  - 加算: +offset
  - 分岐: 新しいサブツリーを生成
  - 回転: 成長方向を変更
  - 色変更: RGB/HSV操作
  - Terminate: 成長停止
操作対象: scaleXYZ, 色, 成長方向, 分岐角度
空間: 3次元（ツリー構造）
```

### 5.3 Hox 遺伝子モデル（位置依存の発現制御）

生物学の Hox 遺伝子（ホメオティック遺伝子）にインスパイア:
- 遺伝子セットは全細胞で同一だが、**位置情報に応じて異なるルールが活性化**
- 「頭側 5 節では Condition 1-3 のみ有効」「尾側では Condition 5-8 が優勢」等
- これにより「前端は大きく、中間は細く、後端は広がる」等の大域的パターンが可能に
